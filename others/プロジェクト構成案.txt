これは **「Claude Code に読ませる知識・ルール・実行制御・サブエージェント・自動化」を階層分離したプロジェクト構成** です。
ざっくり言うと、**CLAUDE.md は全体方針、rules は場所別ルール、skills は作業別手順、agents は専門担当、hooks/settings は強制ルール** です。

---

# ルート直下ファイル

| ファイル               | 役割                 | 記述内容                                          |
| ------------------ | ------------------ | --------------------------------------------- |
| `CLAUDE.md`        | プロジェクト全体の最重要コンテキスト | 目的、技術スタック、開発方針、禁止事項、テスト・ビルドコマンド、レビュー基準を書く     |
| `CLAUDE.local.md`  | 個人・ローカル環境用の補足      | 自分の開発環境、ローカル実行方法、個人用メモを書く。Git管理しない            |
| `AGENTS.md`        | エージェント設計の目次        | どのサブエージェントを、どんな場面で使うかを書く                      |
| `.mcp.json`        | MCPサーバー設定          | GitHub、DB、ブラウザ、ファイル操作など外部ツール連携を書く。秘密情報は直接書かない |
| `.worktreeinclude` | 並列作業用ワークツリー設定      | サブエージェントが別ブランチ・別作業領域で使うべきファイルを書く              |

## `CLAUDE.md`

**常に読み込ませたい共通ルール** を書きます。
肥大化すると精度が落ちるので、200行以内が理想です。

```md
# Project Guide

## Project Goal
このプロジェクトは、〇〇を実現するためのアプリケーションである。

## Tech Stack
- Frontend: Next.js / React
- Backend: Node.js / Hono
- DB: PostgreSQL
- Infra: Cloudflare / Docker

## Commands
- Install: `npm install`
- Dev: `npm run dev`
- Test: `npm test`
- Lint: `npm run lint`
- Build: `npm run build`

## Rules
- 既存仕様を壊さない
- 変更後は必ずテストを実行する
- 推測でAPI仕様を変更しない
- Secretは環境変数で扱う

## Done Definition
- テストが通る
- lintが通る
- 型エラーがない
- 変更理由を説明できる
```

# `.claude/` 配下

`.claude/` は **Claude Code専用の運用レイヤー** です。
ここに、ルール・スキル・エージェント・ワークフロー・フックを集約します。

```txt
.claude/
├── settings.json
├── settings.local.json
├── rules/
├── skills/
├── agents/
├── workflows/
├── agent-memory/
├── hooks/
└── output-styles/
```

---

## `.claude/settings.json`

**チーム共通の強制設定** を書きます。
特に危険コマンドの禁止、hooksの登録、権限制御を担当します。

```json
{
  "permissions": {
    "deny": [
      "rm -rf /",
      "rm -rf *",
      "git push --force",
      "cat .env",
      "cat .env.local"
    ]
  },
  "hooks": {
    "PostToolUse": [
      ".claude/hooks/format.sh"
    ],
    "PreToolUse": [
      ".claude/hooks/protect.sh"
    ]
  }
}
```

---

## `.claude/settings.local.json`

ローカル専用のClaude設定を書きます。
Git管理しません。

```json
{
  "permissions": {
    "allow": [
      "npm test",
      "npm run lint",
      "npm run build"
    ]
  }
}
```

# `.claude/rules/`

`rules/` は **場所別・文脈別の軽量ルール** です。
`CLAUDE.md` に全部詰め込まず、関心ごとごとに分割します。

```txt
.claude/rules/
├── testing.md
├── api-design.md
└── frontend/
    └── react.md
```

画像では `tetting.md` になっていますが、意図としてはおそらく `testing.md` です。

---

## `.claude/rules/testing.md`

テスト方針を書きます。

```md
# Testing Rules

- 機能追加時は必ずテストを追加する
- バグ修正時は再発防止テストを追加する
- 外部APIはモックする
- テストは読みやすさを優先する
- 変更後は `npm test` を実行する
```

---

## `.claude/rules/api-design.md`

API設計ルールを書きます。

```md
# API Design Rules

- REST APIはリソース指向で設計する
- エラー形式は統一する
- バリデーションは入口で行う
- レスポンスには不要な内部情報を含めない
- 破壊的変更は避ける

## Error Format
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid request"
  }
}
```

React / フロントエンドの実装ルールを書きます。

# React Rules

- propsの型を明示する
- UIコンポーネントは責務を小さく保つ

## `.claude/workflows/release-train.js`

リリース前の確認をまとめて実行します。

```js
#!/usr/bin/env node

import { execSync } from "node:child_process";

const commands = [
  "npm test",
  "npm run lint",
  "npm run build"
];

for (const command of commands) {
  console.log(`Running: ${command}`);
  execSync(command, { stdio: "inherit" });
}

console.log("Release checks completed.");
```

# `.claude/hooks/`

`hooks/` は **Claudeの判断に関係なく実行される強制処理** です。
安全性・フォーマット・禁止操作の最後の砦です。

```txt
.claude/hooks/
├── format.sh
└── protect.sh
```

---

## `.claude/hooks/format.sh`

ツール実行後に自動フォーマットします。

```sh
#!/usr/bin/env bash
set -e

echo "Running formatter..."

if [ -f package.json ]; then
  npm run lint --if-present
  npm run format --if-present
fi
```

---

## `.claude/hooks/protect.sh`

危険操作をブロックします。

```sh
#!/usr/bin/env bash
set -e

echo "Running protection check..."

# 例：本番環境ファイルの存在をチェック
if [ -f ".env.production" ]; then
  echo "Warning: .env.production exists. Do not expose secrets."
fi
```

実際には、`rm -rf`、`.env` 読み取り、強制push、本番DB操作などを防ぐ設計にします。

# この構成の分担まとめ

| 層      | 場所                                    | 目的                     |
| ------ | ------------------------------------- | ---------------------- |
| 全体方針   | `CLAUDE.md`                           | 常に読ませるプロジェクト憲法         |
| ローカル補足 | `CLAUDE.local.md`                     | 個人環境メモ                 |
| パス別ルール | `.claude/rules/` / `src/**/CLAUDE.md` | 対象領域だけのルール             |
| 作業手順   | `.claude/skills/`                     | セキュリティレビュー、デプロイ、リリースなど |
| 専門担当   | `.claude/agents/`                     | レビュー、デバッグ、DB確認         |
| 自動実行   | `.claude/hooks/`                      | フォーマット、安全確認、禁止操作       |
| 強制設定   | `.claude/settings.json`               | 権限、hook、実行制御           |
| 知識蓄積   | `.claude/agent-memory/`               | エージェント別の長期メモ           |
| 設計資料   | `docs/`                               | 仕様・設計判断の保存             |
| 補助道具   | `tools/`                              | スクリプト、再利用プロンプト         |

---

最初から全部作るなら、まずはこの **最小セット** で十分です。

```txt
final-boss-project/
├── CLAUDE.md
├── AGENTS.md
├── .mcp.json
├── .claude/
│   ├── settings.json
│   ├── rules/
│   │   ├── api-design.md
│   │   └── testing.md
│   ├── skills/
│   │   ├── security-review/
│   │   │   └── SKILL.md
│   │   └── deploy/
│   │       └── SKILL.md
│   ├── agents/
│   │   ├── code-reviewer.md
│   │   └── debugger.md
│   └── hooks/
│       ├── format.sh
│       └── protect.sh
├── docs/
│   └── architecture.md
└── src/
    ├── api/
    │   └── CLAUDE.md
    └── payment/
        └── CLAUDE.md
```

この構成のキモは、**「Claudeにお願いするルール」と「システム的に強制するルール」を分けること**です。
`CLAUDE.md` や `rules/` は方針、`settings.json` と `hooks/` は実行ガード。ここを分けると、AI開発がかなり安定します。
